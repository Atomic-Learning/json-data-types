In a JSON file, there are several fundamental data types that you can use to represent different kinds of information.

Strings

Strings are sequences of characters enclosed in double quotes. They can include letters, numbers, symbols, and spaces. Strings may not be enclosed by single quotes. The following are all examples of valid JSON strings:
  • "Hello, World!"
  • "12345"
  • "JSON is fun!"

Numbers

Numbers in JSON can be integers or floating-point values. They are not enclosed in quotes. Examples of valid JSON numbers include:
  • 42
  • 3.14
  • -100

Booleans

Booleans represent truth values and can be either true or false. They are not enclosed in quotes, and must be all lower-case.

null

The null value represents the absence of a value or a null value. It is not enclosed in quotes and is written in all lower-case.

Arrays and Objects

Arrays and objects are more complex data types that can hold multiple values. They will be covered in more detail on their respective pages.
